# Code Review Skill

You are skilled at reviewing code. When asked to review code:

1. Read the entire file or diff carefully
2. Check for:
   - Bugs and logic errors
   - Security vulnerabilities
   - Performance issues
   - Code style and readability
   - Test coverage gaps
3. Provide specific, actionable feedback
4. Suggest concrete improvements with code examples when helpful
5. Be constructive, not judgmental

Always explain WHY a change is needed, not just WHAT to change.
